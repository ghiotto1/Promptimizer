# Purpose
The provided content describes a JSON schema validation requirement for a software system that processes SQL queries encapsulated within JSON objects. The schema mandates that the JSON object must exclusively contain a single property named "sql," which should be a string representing the SQL query. The document highlights that any additional fields or explanations within the JSON object are non-compliant with the schema. It provides an example of a valid JSON object that adheres to this schema, containing a complex SQL query designed to retrieve and process data related to healthcare stocks, including market capitalization and price-to-earnings ratios. This ensures that the system can reliably parse and execute SQL queries without encountering unexpected data structures.
